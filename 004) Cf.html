- 시맨틱 태그의 필요성 (p66)

 
      1) 제목,본문의 구분의 명확함
      2) PC나 모바일의 웹 브라우저와 스마트기기의 다양한 화면에서 웹 문서를 표현하기 쉬워짐.

- header 태그 

      : head 영역과는 다르다. 말 그대로 헤더 영역을 표현한다.(Ex) 상단부분)

- nav 태그 

      1) 다른 웹 문서로 연결하는 링크
      2) nav 태그는 웹 문서의 위치에 영향을 받지 않으므로 헤데,푸터 사이드 바 안에 포함 가능하며 또한 독립하여 사용할 수도 있다.
      3) 추가적으로 nav 태그를 여러개 사용할 경우 각각의 id 속성에 따라 다른 스타일을 설정 가능하다.

- main 태그 

      1) 웹 문서에서 핵심이 되는 내용을 넣는다.
      2) main 태그는 웹 문서에서 한번만 사용가능하다.

- article 태그 

      1) 독립된 웹 콘텐츠 항목을 지칭한다.

- section 태그 

      1) 콘텐츠 영역 => 여러개의 콘텐츠를 묶는 용도

- aside 태그 

      : 본문위치에 왼/오른쪽 위치에 내용 추가용도

- footer 태그 

      1) 맨 아래쪽에 있는 영역
      2) 제작정보,저작권,연락처 작성용도
      3) header,section,article 등 다른 시맨틱 태그 모두 사용가능하다.

- div 태그 

      1) 시맨틱 태그가 나오기 전에는 헤더,내비게이션,푸터의 부분을 구별할때에 사용
      2) 현재도 사용중 (div = division)
      3) <div> 태그는 <div id = "header">  or <div class = "detail">
        처럼 id나 class 속성을 사용하여 문서 구조를 만들거나 스타일을 적용할 때 사용한다.
        즉, 영역 구별하거나 스타일로 문서를 꾸미는것을 칭한다.
